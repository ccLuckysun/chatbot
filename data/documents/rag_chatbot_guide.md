# RAG ChatBot 示例知识库

这个项目是一个简单但完整的 RAG 问答助手。它包含文档加载与切片、Embedding 模型、ChromaDB 向量数据库、向量检索器、上下文构建器和 LLM 生成器。

## RAG 工作流程

RAG 是 Retrieval-Augmented Generation 的缩写，意思是检索增强生成。典型流程是：用户提问后，系统先把问题转换成向量，再从向量数据库中检索最相关的文档片段，然后把问题和检索到的上下文一起交给大语言模型生成答案。

这种方式可以让回答更贴近项目自己的资料，并且能在界面中展示参考来源。

## 项目组件

- 文档加载与切片：读取 `data/documents` 下的 `.md` 和 `.txt` 文件，并拆分成适合检索的片段。
- Embedding 模型：把文档片段和用户问题转换为向量。
- 向量数据库：使用 ChromaDB 持久化存储向量和文档元数据。
- 检索器：根据用户问题的向量检索最相关的文档片段。
- 上下文构建：把检索结果整理成 LLM 可以使用的上下文。
- LLM：基于问题和上下文生成中文回答。

## 运行模式

如果 `.env` 中完整配置了 `LLM_*` 和 `EMBEDDING_*`，应用会进入完整 RAG 模式。启动或点击“重建知识库索引”时，系统会读取文档、生成向量并写入 ChromaDB。

如果没有完整配置这些变量，应用会进入本地关键词检索兜底模式。这个模式不调用外部 API，只适合演示文档读取、简单检索和来源展示，不代表完整 RAG 能力。

## 如何扩展知识库

把新的 `.txt` 或 `.md` 文件放入 `data/documents` 目录，然后在网页侧边栏点击“重建知识库索引”。完整 RAG 模式下会重新生成 ChromaDB 向量索引；本地兜底模式下会重新读取文档并更新关键词检索内容。
